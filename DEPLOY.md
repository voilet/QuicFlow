一、 需求文档 (PRD)

1. 业务背景
   为了实现应用自动化的生命周期管理，需要构建一个发布中心。该中心支持应用的版本更迭、故障快速恢复（回滚）以及基于 Docker 的容器化部署监控。

2. 核心功能描述
   发布 (Deploy): 将指定版本的镜像/包部署到目标环境。

升级 (Upgrade): 在现有运行版本基础上，平滑替换为新版本。

回滚 (Rollback): 当新版本出现异常时，一键恢复至上一个稳定的运行版本。

Docker 监控: 实时上报 Docker 容器的 ContainerID、Status（Running, Exited, Paused 等）及健康检查结果。

3. 业务流程与异常处理
   发布流程: 触发发布 -> 拉取镜像 -> 启动容器 -> 健康检查。

自动回滚机制: 若发布过程中“启动失败”或“健康检查超时”，系统必须自动触发回滚指令，重新拉起旧版本容器。
